# programming-tutor

**Claude Code / Cowork 用 Agent-Skill — プログラミング言語とフレームワークを、Claude を家庭教師として Step-by-Step で学ぶためのもの。**

VS Code などで実際に手を動かしながら、小さなハンズオン課題とコードレビューを繰り返して身につけていきます。

🌐 **言語 / Language**: **日本語** · [English](./README.en.md) · [中文](./README.zh.md) · [Español](./README.es.md) · [Français](./README.fr.md)

> スキル本体（Claude への指示）も **日本語 / English / 中文 / Español / Français** の 5 言語に対応しています。学習者の初回メッセージの言語を自動判定し、そのまま学習を進めます。

---

## こんな人のためのスキル

### ✅ 向いているケース

- これからプログラミングを始める、または新しい言語・フレームワークに入門したい
- 独学で進めていて「体系立ったカリキュラム」と「その場でコードを見てくれる人」が欲しい
- 公式ドキュメントはあるけど、どの順番で何から学べば良いか分からない
- 書いたコードを誰かに添削してほしいが、周りに相談できる人がいない
- 忙しくて学習時間が細切れ（1 日 15〜60 分）なので、短く区切った教材が欲しい
- 他言語の経験があり、新しい言語の「流儀」「落とし穴」を効率よく知りたい

### ⚠️ 向いていないケース

- **業務コードそのものを書かせたい** — これは学習用のスキルです。業務用途は別のワークフローを使ってください
- 完成したアプリをすぐ欲しい（学習を飛ばして成果物だけ作りたい）
- 既にその技術を十分使いこなしていて、辞書的な質問だけしたい

## 重要：これは学習用スキルです（業務用ではありません）

このスキルは **「学ぶための環境」** を提供するものです。以下の点に注意してください。

- **仕事の実装そのものは依頼しないでください。** エージェントは意図的に解答を出し渋り、ヒントで導く振る舞いをします。業務で急いでコードを書きたい場合、この挙動は邪魔になります。
- **生成される学習計画・課題・サンプルコードは教育を目的としたものです。** 実務の本番コードとは前提が違います（セキュリティ、性能、エッジケースの網羅は最小限）。そのまま業務コードに転用しないでください。
- **機密情報を入力しないでください。** 社内のプロプライエタリなコード、個人情報、APIキー等は学習のチャット履歴に載せない運用が安全です。
- **何かを決定するときは公式ドキュメントで裏取りしてください。** エージェントのカリキュラムは推奨ルートの一つですが、唯一の正解ではありません。バージョンやエコシステムの事情で最新動向と差分が出ることがあります。

## インストール

このリポジトリには**ソースファイル**が置かれています。インストール方法は 3 通りあります。

### A) GitHub Releases から `.skill` をダウンロード（最も簡単）

1. [Releases ページ](https://github.com/SimJ/programming-tutor-skill/releases) から最新版の `programming-tutor.skill` をダウンロード
2. **Claude Code**: `.skill` ファイルをダブルクリックするか、UI の "Install Skill" で読み込む
3. **Cowork**: `.skill` をチャットにドラッグして表示される "Save Skill" ボタンをクリック

### B) リポジトリを直接クローンして配置

開発者向け。常に最新の `main` を追いかけたい場合はこちらが便利です。

```bash
# Claude Code の場合
git clone https://github.com/SimJ/programming-tutor-skill.git ~/.claude/skills/programming-tutor

# Cowork の場合は、設定画面で表示される skills フォルダの配下にクローンする
```

更新するときは `cd ~/.claude/skills/programming-tutor && git pull` だけで済みます。

### C) 自分でビルドして `.skill` を作る

フォークして改造した版を使いたいときや、Releases が公開されていないときはこちら。

```bash
git clone https://github.com/SimJ/programming-tutor-skill.git
cd programming-tutor-skill
bash scripts/build.sh
# => dist/programming-tutor.skill が作られる
```

以降は A) と同じ要領でインストールします。

## 使い方

### 1. 起動する

次のような発言で自動的にスキルが起動します。わざわざ「このスキルを使って」と言う必要はありません。

- 「Rust を一から学びたい」
- 「Spring Boot で API 作れるようになりたいんだけど」
- 「React のステップバイステップ学習をお願い」
- 「TypeScript のコードレビューして」
- 「プログラミング初めてなんだけど何から始めればいい？」

逆に、明示的に呼びたい場合は「programming-tutor スキルで」と前置きしても OK です。

### 2. ヒアリングに答える

エージェントは最初に 4 つの質問をします（`AskUserQuestion` で多くは選択式）。

- 対象技術（どの言語 / フレームワーク）
- 現在のレベル（未経験 / 他言語経験あり / 基礎あり / 中級以上）
- ゴール（業務 / 趣味 / 資格・面接 / 純粋な興味）
- ペース（1 日の時間、目標完了時期）

これに加えて、環境確認（OS、エディタ、ランタイムのバージョン）を行います。必要であれば `scripts/check_env.sh` を使えます：

```bash
bash scripts/check_env.sh rust     # 指定技術のみ
bash scripts/check_env.sh all      # 全部まとめて
```

### 3. 学習計画を受け取る

ヒアリング結果をもとに、`learning-plan.md` と `progress.md` の 2 ファイルが作業ディレクトリに作成されます。

- `learning-plan.md` — 章立て、到達状態、推奨ツール、最終成果物像
- `progress.md` — ステップごとの進捗チェック、学んだこと、詰まった場所のメモ

計画内容はあなたがレビューして OK/修正を伝えてから次に進みます。

### 4. Step-by-Step ループに入る

各ステップで、エージェントは次のフォーマットで教材を出します。

```
## Step N: <ステップ名>
### なぜ学ぶのか
### コアの考え方
### 最小例
### 今回のあなたの課題
- [ ] やること1
- [ ] やること2
```

ここで **一度止まります**。あなたが VS Code でコードを書く番です。

### 5. 進捗を伝える

エージェントは以下のキーワードで動きます。

| 発言 | 起きること |
| --- | --- |
| 「できた」「終わった」 | 確認してもらえる段階。`progress.md` が更新され、次ステップ案内 |
| 「レビューして」「見て」「確認して」 | コードを精査 → バグ・設計・好みの 3 段階でフィードバック |
| 「ヒント」「詰まった」 | 答えではなくヒントを段階的に出してくれる（質問→関連概念→部分例→最後に解答） |
| 「次へ」 | 次のステップへ |
| 「休憩」 | 現在地をサマリして中断。次回再開しやすく |
| 「飛ばしたい / もっと難しく」 | 難易度調整。レベル再設定 |
| 「別の技術に切り替え」 | 新しい技術用にヒアリングから再スタート |

### 6. コードレビューの形式

レビュー結果はこの構造で返ってきます。

```
👍 良いところ           （具体的に 1〜3 点）
🚨 直したい（バグ系）   （最優先）
⚠️ 改善したい（設計系）  （最大 2 点）
💡 好み・発展           （任意）
次の一歩               （このまま次へ / 直して再提出）
```

初心者には枝葉の指摘は控えめ、中級以上には設計トレードオフまで踏み込むよう設計されています。

## 多言語対応

このスキルは学習者の母語として **日本語 / English / 中文（简体） / Español / Français** の 5 言語をサポートしています。

- エージェントは**初回メッセージの言語**を自動判定して、その言語で教え始めます
- 判定できないときは選択式の質問を 1 回だけ行います
- 会話中いつでも切り替え可能です。例: 「今度は英語で」「请用中文」「en español por favor」「passons au français」「日本語に戻して」
- コード・ファイル名・コマンド・ライブラリ名は**どの言語を選んでも英語のまま**です

翻訳版の README は、このディレクトリ直下の `README.en.md` / `README.zh.md` / `README.es.md` / `README.fr.md` にあります。

## カスタマイズ

### 対応技術を増やす

新しい技術（例：Next.js）を追加するときは：

1. `references/nextjs.md` を作る。既存ファイル（特に `react.md` や `nestjs.md`）を雛形にすると手早い
2. 含めると良いセクション：
   - 前提確認（必要ランタイム、バージョン目安）
   - カリキュラム（章立て・到達状態・ステップ一覧）
   - 典型的な落とし穴
   - コードレビュー観点
   - 推奨成果物
3. `SKILL.md` の「Planning」セクションのリファレンス一覧に追記する

### 教え方の調整

- ステップの粒度を変えたい → `references/pedagogy.md` の「ステップサイズのチェックリスト」を編集
- レビューの厳しさを変えたい → `references/code_review.md` の「レベルに応じて深さを変える」節を編集
- 日本語 / 英語の使い分けを変えたい → `SKILL.md` の「User-facing language」節を編集

### 進捗ファイルの場所

既定では作業ディレクトリのルートに `learning-plan.md` と `progress.md` を作成します。プロジェクトの中で作りたい場合は、エージェントに「`./docs/` 配下に作って」と言えば従います。

## FAQ

**Q. カリキュラムを勝手に進めたい。エージェントの確認待ちが煩わしい。**
A. 「自走モードで」「一気に章 3 まで進めて」などと言えば、確認ステップを圧縮します。ただし自走モードは学習定着が下がるのでおすすめはしません。

**Q. エージェントが答えを教えてくれない。**
A. 仕様です。2〜3 回のヒント交換を経て、それでも詰まっている場合は完全な解答を解説付きで出します。早く解答が欲しい時は「解答を見せて」と直接言えば出します。

**Q. 途中で対象技術を変えたい。**
A. 「今度は React に切り替え」と言えば、新技術用にヒアリングから再スタートします。前の技術の `progress.md` はそのまま残ります。

**Q. 長期プロジェクト（数週間）で使うには？**
A. 毎セッションの最初に「前回の続き」と言えば、`progress.md` を読んで現在地を把握します。作業ディレクトリを固定しておくのが大事です。

**Q. 業務で急いで実装したい。**
A. このスキルは使わず、通常の Claude / Claude Code でコード生成を依頼する方が適しています。本スキルは意図的に「手を動かさせる」設計です。

**Q. コードが長すぎてレビューしきれない。**
A. 1 ファイル 500 行を超えるコードはレビューが散漫になるため、エージェントが「先にどこを見てほしいか」を聞き返します。学習中はそもそも 1 ステップで書くコード量が過大にならないよう設計されているので、通常は起きません。

**Q. トリガーされない。**
A. SKILL.md の description を編集して、あなたがよく使う言い回し（例：「◯◯入門」）を追加してください。それでもうまく起動しない場合、スキル名を直接呼ぶ（「programming-tutor で」）運用も可能です。

## クレジット

このスキルは Anthropic の `skill-creator` の作法に従って構築されています。
構造（`SKILL.md` + `references/` + `assets/` + `scripts/`）は progressive disclosure パターンを採用しており、必要な情報だけがエージェントのコンテキストにロードされます。
