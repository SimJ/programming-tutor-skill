# Rust — 学習カリキュラム

対象: Rust を最初から学び、所有権を理解し、小さな CLI を書けるようになるところまで。
Rust は他言語と比べて学習曲線が急なので、ステップを細かく刻む。

## 前提確認

- `rustup` インストール（`rustup --version`, `rustc --version`, `cargo --version`）
- VS Code + `rust-analyzer` 拡張
- C や C++ のようなメモリ管理の概念を学習者が知っているかで、所有権の説明の深さを変える

## カリキュラム（全 8 章、他言語より 1 章多め）

- [1.2] `main.rs` を読む、`cargo run`

**到達状態**: コンパイルエラー "borrow of moved value" を見て原因を言語化できる

## 典型的な落とし穴

- "borrowed value does not live long enough" → ライフタイムではなく、そもそも設計（所有権の流れ）を見直す
- `String` と `&str` の混同
- `Vec` を for で回しながら中身を変更しようとする
- `.clone()` を乱用して学習を進める → 最初は OK だが、章 3 が終わったら「なぜ clone せずに済むか」を意識
- `unwrap` 連発 → `?` への切替タイミングを教える
- async / await は**このカリキュラムの範囲外**。別カリキュラムに分ける（tokio は学習曲線が別）

## コードレビュー観点

- 所有権の流れが自然か（無駄な `clone` がないか）
- `Option` / `Result` を握り潰していないか（`unwrap` 連打）
- エラー型がドメインを表しているか
- `pub` の範囲が必要最小限か
- `cargo clippy` の警告を取っているか
