# Language General — 言語を問わない学習雛形

Python / JavaScript / TypeScript / Java / C# / Go / Ruby など、汎用的な言語を学ぶときの**章立てテンプレート**。
技術別ファイルがない言語を学びたい学習者にはこれをベースにする。

## 前提確認

- ランタイムが入っているか
- エディタ（VS Code 想定）
- パッケージマネージャ（pip / npm / cargo / dotnet / maven / gradle 等）

## カリキュラム雛形（全 6 章）

### 第 1 章 環境構築と Hello World
- [1.1] ランタイムのインストール確認
- [1.2] 最初のファイルを作って実行
- [1.3] コメント、出力、簡単な算術

**到達状態**: ターミナルでファイルを実行して "Hello" が出せる

### 第 2 章 基礎構文
- [2.1] 変数と基本型（数値、文字列、真偽値）
- [2.2] 条件分岐（if / else）
- [2.3] 繰り返し（for / while）
- [2.4] 配列/リストと基本操作
- [2.5] 辞書/マップ

**到達状態**: FizzBuzz を書ける。与えられた配列の合計・最大を求められる

### 第 3 章 関数とモジュール
- [3.1] 関数の定義、引数、戻り値
- [3.2] スコープと変数の寿命
- [3.3] ファイル分割とモジュールのインポート
- [3.4] ドキュメンテーション（docstring / JSDoc / XML コメント等）

**到達状態**: 複数ファイルに分けたプロジェクト構造でコードを書ける

### 第 4 章 データ構造と抽象化
- [4.1] 構造体 / クラス / レコード（その言語の主要な抽象手段）
- [4.2] メソッドと状態
- [4.3] 継承 or 合成 or インターフェース（その言語の流儀に合わせて）
- [4.4] コレクション操作（map / filter / reduce 系）

**到達状態**: 簡単なドメインを型/クラスでモデリングできる

### 第 5 章 エラー処理・テスト・デバッグ
- [5.1] エラー/例外の基本（try/catch or Result 型など）
- [5.2] ログとデバッグ（print/logger, デバッガの使い方）
- [5.3] 標準テストフレームワークで最初のテストを書く
- [5.4] GitHub など VCS の最低限（init / add / commit）

**到達状態**: 失敗ケースを含む関数にテストを書いて緑にできる

### 第 6 章 小さな成果物を作る
- [6.1] プロジェクトの設計（ファイル構成、責務分割）
- [6.2] 実装（CLI ツール、簡易 API、データ処理スクリプト等）
- [6.3] テストを追加
- [6.4] README を書いて公開準備

**到達状態**: 一つの完結したプロジェクトを自分で書けた

## 成果物アイデア（最後の章で作るもの）

- CLI Todo アプリ（ファイルに保存）
- ユーザーからの入力に答える簡単な数当てゲーム
- CSV を読んで集計する小スクリプト
- 簡易 Web スクレイパー（単一サイトの見出し取得）
- 天気 API を叩いて現在の気温を表示する CLI

ゴールが業務寄りなら API クライアント系、趣味寄りならゲーム系を薦める。

## 言語ごとの軽い注釈

### Python
- インデント強制 / PEP 8 / `list comprehension` / `f-string`
- `venv` または `poetry` を第 1 章で導入
- テストは `pytest`

### JavaScript / TypeScript
- `let` / `const` 徹底、`var` は使わない
- Node.js で実行、`package.json`
- TypeScript を選ぶなら第 2 章の時点で `tsc` を導入
- テストは `vitest` or `jest`

### Java
- IDE は IntelliJ か VS Code + Java Extension Pack
- ビルドは Gradle または Maven
- テストは JUnit 5

### Go
- `go mod init`
- フォーマッタ `gofmt` は学習初期から習慣に
- テストは標準 `testing`

### Ruby
- `bundle init` で Gemfile 管理
- テストは RSpec か Minitest
